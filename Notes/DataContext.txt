Контекст даних

Контекст (взагалі) - оточення, контейнер

Контекст даних - це елемент (як правило, об'єкт), що є оточенням для сутностей
(для даних)

Контекст програми (застосунку) - AppContext - відомості про оточення програми/запуску:
папки (базова папка, робоча папка, ресурсна папка), цільова платформа

Контекст встановлення/розгортання (Host, Deploy) - відомості про оточення
(як AppContext) у місці встановлення/розгортання

Контест даних рекомендується виконувати за концепцією Singleton (щоб існував
лише один об'єкт контексту) та загальнодоступності.
У WPF один з варіантів - використати спільний об'єкт App, або впроваджувати
контейнер залежностей
App {											   | У будь-якому вікні
	private static DataContext _dataContext ;	   | можна звернутись до 
	public static DataContext DataContext {		   | контексту даних як
		get {									   | App.DataContext
			if( _dataContext is null ) {		   | 
				_dataContext = new() ;			   | Перше звернення створить
			}									   | new() об'єкт, інші будуть
			return _dataContext ;				   | отримувати посилання на
		}										   | раніше створений об'єкт
	}
}
